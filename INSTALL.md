# Автономный установщик CursorAI Autonomous Extension

## Описание

Автономный установщик автоматически выполняет все необходимые шаги для сборки и установки расширения в CursorAI.

## Использование

### Windows

Просто запустите:
```bash
install.bat
```

Или дважды кликните на файл `install.bat` в проводнике Windows.

### Linux / macOS

1. Сделайте скрипт исполняемым:
   ```bash
   chmod +x install.sh
   ```

2. Запустите:
   ```bash
   ./install.sh
   ```

### Универсальный способ (Node.js)

Если у вас установлен Node.js, вы можете использовать универсальный скрипт:

```bash
npm run install
```

или напрямую:

```bash
node install.js
```

## Что делает установщик

1. **Проверка зависимостей**
   - Проверяет наличие Node.js (версия 18+)
   - Проверяет наличие npm

2. **Установка зависимостей проекта**
   - Запускает `npm install`
   - Устанавливает все необходимые пакеты

3. **Компиляция TypeScript**
   - Компилирует исходный код в JavaScript
   - Проверяет успешность компиляции

4. **Сборка расширения**
   - Устанавливает vsce (если необходимо)
   - Создает .vsix файл для установки

5. **Установка в CursorAI**
   - Автоматически находит CursorAI
   - Устанавливает расширение
   - Если CursorAI не найден, предоставляет инструкции для ручной установки

## Требования

- **Node.js** версии 18 или выше
- **npm** (устанавливается вместе с Node.js)
- **CursorAI** (для автоматической установки)

## Решение проблем

### Ошибка: "Node.js не найден"

Установите Node.js с официального сайта: https://nodejs.org/

### Ошибка: "CursorAI не найден"

Установщик предоставит инструкции для ручной установки. Вы также можете:

1. Установить CursorAI
2. Запустить установщик снова

Или установить вручную:
1. Откройте CursorAI
2. Нажмите `Ctrl+Shift+X` (или `Cmd+Shift+X` на macOS)
3. Нажмите на `...` вверху панели расширений
4. Выберите "Install from VSIX..."
5. Выберите созданный .vsix файл

### Ошибка при компиляции

Убедитесь, что:
- Node.js версии 18 или выше
- Все зависимости установлены (`npm install`)
- Нет синтаксических ошибок в коде

### Ошибка при сборке .vsix

Установщик попытается установить vsce автоматически. Если это не удалось:

```bash
npm install -g @vscode/vsce
```

Затем запустите установщик снова.
