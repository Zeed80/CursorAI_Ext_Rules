# Объяснение предупреждений VS Code

## Предупреждения о паттернах настроек

### Предупреждения, которые можно игнорировать:

```
WARN Settings pattern "chat.*" doesn't match any settings
WARN Settings pattern "mcp" doesn't match any settings
WARN Settings pattern "GitHub.copilot.manageExtension" doesn't match any settings
```

**Причина**: Эти предупреждения **НЕ связаны с нашим расширением**. Они появляются из-за того, что:
- Другие расширения VS Code пытаются использовать паттерны настроек, которые не определены
- Сам Cursor IDE использует эти паттерны для своих внутренних настроек
- VS Code проверяет все паттерны настроек при загрузке расширений

**Решение**: Эти предупреждения можно безопасно игнорировать. Они не влияют на работу нашего расширения.

### Сообщения о моделях

```
[Extension Host] No models found in CursorAI. Models will be selected automatically by CursorAI.
[Extension Host] This is normal if using Cursor AI - it will auto-select models.
```

**Причина**: Это **нормальное поведение**. Если модели не найдены через API или настройки, CursorAI автоматически выберет подходящую модель для каждого агента.

**Решение**: Это ожидаемое поведение. Расширение работает корректно, используя автоматический выбор моделей CursorAI.

### Ошибки, не связанные с нашим расширением

```
ERR [Extension Host] [otel.error] OTLPExporterError: Bad Request
ERR Failed to load team commands: ConnectError: [unauthenticated]
```

**Причина**: Эти ошибки связаны с:
- OpenTelemetry (система телеметрии Cursor IDE)
- Team commands (функции командной работы Cursor IDE)

**Решение**: Эти ошибки не связаны с нашим расширением и не влияют на его работу.

## Как проверить, что расширение работает правильно

### 1. Проверка инициализации

Откройте консоль разработчика (Help → Toggle Developer Tools) и проверьте:

```
[Extension Host] CursorAI Autonomous Extension is now active!
[Extension Host] CursorAPI initialized with API key, version: auto
```

### 2. Проверка агентов

1. Откройте панель "Агенты" (CursorAI в боковой панели)
2. Убедитесь, что все агенты отображаются
3. Проверьте статус агентов (должен быть "Ожидает" или "Работает")

### 3. Проверка назначения моделей

1. Правой кнопкой мыши на агенте → "Select Model for Agent"
2. Выберите модель или "Автоматический выбор"
3. Проверьте, что модель сохранилась (в описании агента должна появиться информация о модели)

### 4. Проверка настроек

1. Откройте `settings.json` (Ctrl+Shift+P → "Preferences: Open User Settings (JSON)")
2. Найдите `cursor-autonomous.agents`
3. Убедитесь, что сохранен только `selectedModelId` (строка), а не объект

## Если что-то не работает

### Проблема: Агенты не инициализируются

**Решение**:
1. Перезапустите VS Code
2. Проверьте консоль разработчика на наличие ошибок
3. Убедитесь, что расширение активировано

### Проблема: Модели не назначаются

**Решение**:
1. Проверьте, что API ключ CursorAI настроен (если требуется)
2. Проверьте, что в настройках сохранен только `selectedModelId` (строка)
3. Перезапустите VS Code после изменения настроек

### Проблема: Ошибки в консоли

**Решение**:
1. Проверьте, связана ли ошибка с нашим расширением (ищите "CursorAI Autonomous" или "cursor-autonomous" в сообщении)
2. Если ошибка не связана с нашим расширением, её можно игнорировать
3. Если ошибка связана с нашим расширением, сообщите об этом с полным текстом ошибки

## Логирование для отладки

Для включения подробного логирования добавьте в `settings.json`:

```json
{
  "cursor-autonomous.enableVirtualUser": false,
  "cursor-autonomous.autoImprove": true
}
```

В консоли разработчика будут видны все сообщения расширения с префиксом `[Extension Host]`.

## Итог

**Все предупреждения о паттернах настроек (`chat.*`, `mcp`, `GitHub.copilot`) можно безопасно игнорировать** - они не связаны с нашим расширением и не влияют на его работу.

Если расширение работает (агенты отображаются, модели назначаются), значит всё в порядке!
